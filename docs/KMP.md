# KMP.md

Guidance for working in the `shared/` Kotlin Multiplatform module.

## Project Setup

- **Project name**: BazaarShared
- **Root package**: `com.bazaar.shared`
- **Kotlin version**: 2.3.21
- **AGP version**: 9.0.1
- **Gradle wrapper**: included (`gradlew`)
- **Version catalog**: `gradle/libs.versions.toml` - all dependency versions live here, never hardcode versions in build files

Created via Android Studio KMP wizard. The project lives at `bazaar_proj/shared/` and is a standalone Gradle project (not a submodule of the root).

## Modules

The wizard generated four modules. Two are ours, two are scaffolding we keep but don't touch.

### `sharedLogic` - our primary module

Pure Kotlin business logic shared across **all** platforms (iOS buyer app, Android app).

- Targets: `iosArm64`, `iosSimulatorArm64`, `androidLibrary`
- Produces: `SharedLogic.xcframework` (for iOS), AAR (for Android)
- Source sets: `commonMain`, `iosMain`, `androidMain`, `commonTest`, `iosTest`, `androidHostTest`
- Framework base name: `SharedLogic` (set in `build.gradle.kts`)
- Framework is **static** (`isStatic = true`)

Contains:
- Domain models (`@Serializable` data classes)
- Network layer (Ktor client)
- Repositories (AuthRepository, ProductRepository, etc.)
- Validation rules
- Token storage (expect/actual - Keychain on iOS, EncryptedSharedPrefs on Android)
- Error types

**iOS apps link only this framework. `sharedUI` is never added to iOS Xcode targets.**

### `sharedUI` - Android Compose UI

Compose Multiplatform UI components used by the **Android app only**.

- Targets: `androidLibrary` only (no iOS target - iOS uses SwiftUI)
- Depends on `sharedLogic` via `api(projects.sharedLogic)`
- Contains: ViewModels, Compose screens, UI components, navigation
- iOS buyer app (`BazaarApp`) and admin app (`BazaarAdmin`) never use this module - they are pure SwiftUI

### `iosApp` - wizard scaffold

Sample iOS app generated by the wizard. **We do not use this.** Our iOS apps live in `bazaar_proj/ios/`. Leave this untouched - deleting it may require `settings.gradle.kts` changes.

### `androidApp` - wizard scaffold

Sample Android app generated by the wizard. **We do not use this yet.** The real Android app will be built in Phase 5 using `sharedUI` + `sharedLogic`. Leave untouched for now.

## Key Architecture Decisions

### expect/actual for TokenStorage
`TokenStorage` is declared as `expect class` in `commonMain`. Platform implementations:
- **iOS** (`iosMain`): Keychain via `platform.Security` framework. Uses `kSecAttrAccessibleWhenUnlockedThisDeviceOnly`.
- **Android** (`androidMain`): `EncryptedSharedPreferences` (Phase 5 - stub in Phase 1).
- **Tests** (`commonTest`): `InMemoryTokenStorage` - plain `MutableMap`, no platform dependencies.

### BazaarAuth Ktor plugin
A custom `ClientPlugin` that attaches the Bearer token to every outgoing request via `onRequest`. The 401 refresh+retry is handled at the repository layer using a `Mutex` to prevent concurrent refresh races.

### Ktor engine per platform
- **iOS**: `Darwin` engine (wraps `NSURLSession`, zero extra dependencies)
- **Android**: `OkHttp` engine

The engine is injected via expect/actual in `BazaarHttpClient.kt`, not hardcoded.

### Repository pattern
All network access goes through repository interfaces defined in `commonMain`. The implementations are also in `commonMain` - only the engine and storage differ per platform. Swift and Android never call Ktor directly; they call repository functions.

### KMP suspend → Swift async/await
KMP `suspend` functions compile to Swift `async` functions automatically. No callbacks needed. Swift calls:
```swift
let tokens = try await authRepo.login(email: email, password: password)
```

### sharedUI is Android-only
`sharedUI` has no iOS targets. It uses Compose Multiplatform for Android's UI. iOS apps are SwiftUI only. Never add `sharedUI` to an iOS Xcode target.

## Commands

# Run shared tests (runs on JVM - fast)

## Adding a New Dependency

1. Add the version to `gradle/libs.versions.toml` under `[versions]`
2. Add the library reference under `[libraries]`
3. Add a plugin reference under `[plugins]` if needed
4. Reference it in `sharedLogic/build.gradle.kts` inside the correct source set

Example - adding Ktor:
```toml
# libs.versions.toml
[versions]
ktor = "3.1.3"

[libraries]
ktor-client-core = { module = "io.ktor:ktor-client-core", version.ref = "ktor" }
ktor-client-darwin = { module = "io.ktor:ktor-client-darwin", version.ref = "ktor" }
```

```kotlin
// sharedLogic/build.gradle.kts
sourceSets {
    commonMain.dependencies {
        implementation(libs.ktor.client.core)
    }
    iosMain.dependencies {
        implementation(libs.ktor.client.darwin)
    }
}
```

## XCFramework Integration (iOS)

After running `assembleXCFramework`:

1. Output is at: `sharedLogic/build/XCFrameworks/release/SharedLogic.xcframework`
2. In Xcode, drag `SharedLogic.xcframework` into the `BazaarApp` target - **Embed & Sign**
3. Do **not** add it to `BazaarAdmin` - admin app is pure SwiftUI with URLSession, no KMP
4. Import in Swift: `import SharedLogic`

For development, the debug build is at: `sharedLogic/build/XCFrameworks/debug/SharedLogic.xcframework`
